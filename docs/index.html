About ID3dev.

ID3dev is a low-level library for reading, manipulating, and writing ID3 metadata of any version.

As of right now, ID3dev is going through a rewrite to ensure it has more modules and removes a lot of spaghetti code. As of right now, the only completed components are the projects linked list API and buffer streaming API along, with everything to do with ID3v1 and Id3v1.1. ID3v1

ID3v1 Sturcture

ID3v1 is a 128-byte metadata container primarily used with MP3 files to store basic catalog information. This initial release can store artist name, song title, album title, release year, user comment, and genre. ID3v1 is identified by the 3-byte magic number 0x54 0x41 0x47 or in ASCII, TAG, and is located at 128 bytes from the end of an MP3 file using this form of metadata. Following the identifier are the following fields in order song title, artist, album, year, comment, and genere.

ID3v1 is made up of 5 text frames with the Latin1 (ISO-8859-1) character set and 1 integer field. The text fields consist of 4 30 byte fields which are title, artist, album, and comment with 1 4 byte field consisting of the year. All text fields are followed by a single genre byte with all compatible genres listed here.

ID3v1.1 Structure

ID3v1.1 follows the exact same structure as the previous version except the comment field is shortend to make room for an extra integer. In this version the comment field is truncated by 2 bytes making its capacity now 28 bytes. The first byte (ex 29th byte) is used to store a null character to indicate that the following character contains a track number.
